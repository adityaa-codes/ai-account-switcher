# v0.2.0 Release Notes

**Release date:** 2026-03-07

## Highlights

v0.2.0 brings five user-facing features that make day-to-day multi-account workflows significantly faster: a live quota dashboard, an interactive numbered menu, a `/change`-parity command, `pool` subcommand aliases, and a crash-safe handoff flag that eliminates a redundant API round-trip after auto-rotation. Test coverage also improved from 85 % to 91 % in this release.

---

## New Features

### Live Quota Display (`switcher gemini quota`)

Shows remaining quota for every Gemini OAuth profile at a glance, without leaving the terminal:

```
ℹ️  Fetching quota for 2 OAuth profile(s)...

  ──────────────────────────────────────────────────────────────

  work ●
  (you@work.com)
    gemini-2.0-flash          ████████████░░  85.3%

  personal
  (you@personal.com)
    gemini-2.0-flash          ██░░░░░░░░░░░░  12.1% ⚠️  resets in 4 h
```

- Visual progress bar + percentage for each quota bucket
- `●` marks the currently active profile
- `⚠️` warning when remaining quota is below 20 %
- Reset time shown when available
- Only OAuth profiles are checked (API key profiles expose no quota API)

### Interactive Menu (`switcher gemini menu` / `switcher codex menu`)

A numbered TUI that lets you manage profiles without memorising commands:

```
  Gemini Profile Manager
  Active: work
  ────────────────────────────────────
   1)  List all profiles
   2)  Switch to a profile
   3)  Rotate to next profile
   4)  Show live quota usage
   5)  Check profile health
   6)  Add a new profile
   7)  Import credentials
   8)  Toggle auto-rotate on/off
   q)  Quit

  Choice:
```

- Falls back to a compact command reference when stdin is not a TTY (piped, CI, scripts)
- All menu actions delegate to the same CLI handlers, so behaviour is identical

### `/change` Parity Command (`switcher gemini change` / `switcher codex change`)

Mirrors the Gemini CLI `/change` slash command with smarter routing:

| Invocation | Result |
|---|---|
| `switcher gemini change` | Rotate to next profile (same as `next`) |
| `switcher gemini change next` | Rotate to next profile |
| `switcher gemini change work` | Switch to profile labelled `work` |
| `switcher gemini change 2` | Switch to profile at index 2 |

This also updates the installed `/change` slash command so it passes through user-supplied arguments.

### `pool` Subcommand Aliases

`pool` is an alias namespace for users who think of profiles as a pool:

```bash
switcher gemini pool            # → list
switcher gemini pool add        # → add
switcher gemini pool remove     # → remove
switcher gemini pool import     # → import
```

All arguments are forwarded unchanged to the canonical commands.

### Crash-Safe Hook Handoff Flag

When the AfterAgent hook successfully rotates to a new profile, it now writes a short-lived flag file (`~/.config/ai-account-switcher/state/quota_error_gemini.json`) with a 120-second TTL. The BeforeAgent hook reads and clears this flag on its next invocation, skipping the proactive quota API call entirely — eliminating a round-trip to Google's API right after a rotation.

This prevents a race condition where the new profile's quota wasn't yet reflected in the API response, which could cause a false positive and an unintended double-rotation.

### `restart_on_switch` Config Option

```toml
[auto_rotate]
restart_on_switch = false   # set to true to ask Gemini CLI to restart the agent
```

When enabled, the AfterAgent hook appends a note to `systemMessage` requesting an agent restart after rotation, giving Gemini CLI the opportunity to reload its context cleanly.

## Improvements

### Centralised Quota Error Patterns

Quota error detection logic was consolidated from a local list in `gemini_after_agent.py` into a dedicated `switcher/hooks/quota_patterns.py` module. The pattern set was also expanded from 8 to 14 patterns, adding:

- Structured JSON error detection (`"code": 429`, `"status": "RESOURCE_EXHAUSTED"`)
- Additional phrases: `quota.*exhausted`, `free tier limit`, `daily.*limit`
- More precise HTTP status matching (`\b429\b` word-boundary)

Both hooks now import from this shared module, ensuring consistent detection.
