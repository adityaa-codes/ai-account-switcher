# v0.4.0 Release Notes

**Release date:** 2026-03-20

## Highlights

v0.4.0 focuses on true day-one usability for account switching. It adds a plug-and-play setup/adoption workflow, introduces direct recovery commands for broken OAuth/env states, and hardens switching internals for safer concurrent and edge-case behavior. This release also significantly expands failure-path test coverage to improve reliability in real-world, non-ideal environments.

---

## Bug Fixes

### 🔧 Codex env alias consistency

Codex auth-mode conflicts could persist when only one environment alias was handled. The switcher now treats `OPENAI_API_KEY` and `CODEX_API_KEY` consistently across export, preservation, conflict diagnosis, and cleanup paths.

### 🔧 Safer cache and lock handling

Gemini cache cleanup now removes both OAuth cache files used by newer workflows, and switch/repair operations are now guarded with lock files to reduce race-condition risks under concurrent commands.

---

## New Features

### ✨ Plug-and-play setup and discovery

You can now bootstrap quickly using:

```bash
switcher setup
switcher discover
switcher use gemini
switcher use codex
```

This path is designed for users who already logged into Gemini/Codex and want immediate profile adoption.

### 🔑 Diagnostics and remediation workflow

`doctor` and `fix` now form a practical recovery pair for sign-in and env override issues:

```bash
switcher doctor
switcher fix
```

Diagnostics now include wrong symlink target detection for active OAuth profiles, and remediation clears stale API-key overrides safely.

### 🛡️ Acceptance smoke script

A local smoke script is now included for quick validation of setup, adoption, and remediation behavior:

```bash
bash scripts/acceptance-smoke.sh
```

---

## Improvements

### Conservative unknown-health switching

`switcher use` no longer auto-switches to profiles with `unknown` health by default. Operators can still opt in with `--allow-unknown` when needed.

### Keyring-aware discovery guidance

When file credentials are missing but keyring-backed auth is likely, discovery now surfaces actionable guidance instead of only reporting “not found.”

---

## Breaking Changes

None. Existing commands remain available and compatible.
